Things I Did

May 2019 - Aug. 2019

A significant project of mine was not actually directly involved with Yahoo! Mail, but was designing and delivering a feature flag diagnostic tool for my team (and potentially other teams too).

Problem: Feature enablement in the app is managed via a settings file. This is hard to read, and harder to interpret using mere human eyesight. Figuring out whether a feature would be enabled or disabled for a given user configuration can be unpleasant. It is also inconvenient to track the evolution of a feature’s enablement settings over time — jumping wildly through commit history trying to find a feature flag change that broke something is not an ideal way to spend one's Friday evening.

Solution: Leveraging Github's GraphQL API, settings files were fetched and parsed to generate diffs of their states over time. We'd know which ones to fetch by also parsing changelog files and using those version tags to find the relevant settings file versions. A web app was created using React Redux, and showed a timeline view of feature flag setting changes. Users could also filter by flag, enabled/disabled, and user settings.
